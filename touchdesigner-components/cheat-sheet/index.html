Cheat Sheet

TD Pseudomathematical Notation

This is where I will be gathering a user guide / manual / cheat sheet of & for the pseudo mathematical notation I’ll be using to describe image processing concepts.

General Notation:

Ι —

Image:
Represents the input image currently being processed.
Depending on the context, Ι can refer to binary, grayscale, or color images.

ι —

Pixel:
Represents a single pixel within an image. When paired with an image symbol (e.g., Ι ( ι )), it denotes the value of that pixel at position ι. Depending on context, this may refer to a binary state, an intensity value in a grayscale image, or a vector of channel values in a color image.

Edge Detection:

g —

Gradient:
Represents the image gradient, which measures the change in pixel intensity between neighboring pixels.
Subscripts are used to specify defining attributes of the gradient in question, such as the orientation:
gx, gy, gz are gradients along the x, y, & z axis respectively,
gN, gNE... are gradients oriented towards compass directions.
Whether a gradient is a composite of multiple gradients:
gmax, gavg... are the maximum or average response from multiple gradients.
Other attributes for example:
gint, gext... which refer to the interior or exterior gradients of an object’s boundaries...

G —

Gradient Magnitude:
The overall strength of the gradient at a pixel, calculated using the Pythagorean theorem and the images horizontal and vertical gradients.
Ι G = g x 2 + g y 2

Θ —

Edge Orientation:
The direction of the gradient relative to the x-axis, indicating the angle or orientation of an edge. It is computed using the arctangent function and the images horizontal and vertical gradients.
Ι Θ = atan2 ( g y , g x )

k —

Kernel:
A matrix applied to an image during convolution, used for tasks such as blurring, sharpening, embossing, and edge detection. Each pixel in the output image is a function of the nearby pixels in the input image, and the kernel defines how this function operates.
Below is the NE-oriented Kirsch Operator kernel as an example. Notice the subscript notation mirrors that used in the gradient (g) notation:
k NE = [ -3 5 5 -3 0 5 -3 -3 -3 ]

R —

Response:
In symmetrical convolutional operations that do not result in an image gradient, R represents the maximum response from all directional kernels.

Histogram:

p,q

Histograms:
Lowercase letters denote histograms as statistical distributions. Individual bin values are indexed as pi, qi, representing the frequency (or normalized frequency) of intensity value i.

P,Q

Cumulative Distribution Functions (CDFs):
Uppercase letters denote cumulative distributions derived from histograms. Individual cumulative values are indexed as Pi, Qi, representing the cumulative proportion of pixels up to bin i.

Morphology:

⊕ —

Dilation:
Expands the boundaries of foreground regions (typically white pixels) in an image.

δ —

Dilated Image:
The resulting image after applying dilation. Objects are enlarged, gaps are filled, and disjointed components are connected based on the shape and size of the structuring element.

⊖ —

Erosion:
Contracts the boundaries of foreground regions (typically white pixels) in an image

ε —

Eroded Image:
The resulting image after applying erosion. Objects are shrunk, and small details or noise are removed, leaving refined and separated elements.

γ —

Opened Image:
The resulting image after having been opened which involves first applying erosion followed by dilation.
Ιγ= (Ι⊖s) ⊕s

ɸ —

Closed Image:
The resulting image after having been closed which involves first applying dilation followed by erosion.
Ιɸ= (Ι⊕s) ⊖s

⊙ —

Hit‐or‐Miss:
Detects specific patterns of pixels in an image by comparing foreground and background pixels to a composite structuring element ( S ).

η —

Hit‐or‐Miss Result:
The output of the Hit‐or‐Miss transformation. Pixels are active only where the specified composite structuring element’s pattern matches the input image exactly.
Ιη= (Ι⊙s)

s —

Structuring Element:
A predefined shape used in morphological operations to probe the image. The shape and size of the structuring element determine how these morphological operations interact with the objects in the input image.
Much like a convolutional kernel, structuring elements are defined as matrices, but a key difference is that cells can be designated as “don’t cares” (marked with x’s), which are skipped during the morphological transformation. For example:
s = [ x 1 x 0 1 1 0 0 x ]

S —

Composite Structuring Element:
A collection of structuring element, typically used in the Hit‐or‐Miss transformation, that usually include multiple rotations or variations of a base structuring element. This ensures that patterns can be detected regardless of their orientation within the input image. For example, in a Concave Corner mode:
S = { s 1 , s 2 , s 3 , s 4 }
Where
s 1 = [ 0 1 1 x 1 1 x x 0 ] , s 2 = [ x x 0 x 1 1 0 1 1 ] , s 3 = [ 0 x x 1 1 x 1 1 0 ] , & s 4 = [ 1 1 0 1 1 x 0 x x ]

SDFs:

d —

Distance Metric:
Represents the metric used to calculate distances between points. Depending on the context, this may be Euclidean, Manhattan, Chebyshev, or Minkowski.

∂ —

Boundary:
Denotes the boundary of a shape or subset. In the context of SDFs, ∂Ω represents the dividing line between interior and exterior regions — the reference from which signed distances are measured.

Ω —

Subset:
epresents a region or set of pixels within the image space. In SDFs, Ω is the shape or area of interest, while its complement is the space outside the shape.
